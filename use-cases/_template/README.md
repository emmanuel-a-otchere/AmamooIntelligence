# Use Case Scaffold — `<NAME>`

This is a copy-and-fill template. After cloning this folder to `use-cases/<your-name>/`:

1. **Rename the folder** to your use case (`mv use-cases/_template use-cases/<your-name>`).
2. **Fill in `SPEC.md`** — the goal, success criteria, what's general vs specialized, sync strategy.
3. **Add an `OVERLAY.md`** — every file your fork changes/adds on top of OtchereJarvis, with a one-line rationale each.
4. **Add a `tests/` subfolder** with use-case-specific tests.
5. **Update the top-level `use-cases/README.md`** to link your new entry.

See `docs/SPECIALIZING.md` for the full workflow.
